# Step 9：认证鉴权（JWT + 密码加密）

这一步给项目装上「**你是谁**（认证 / Authentication）」和「**你能不能访问**（鉴权 / Authorization）」两项后端最核心的能力：用户可以注册、登录，登录后拿到一张「身份凭证」，之后带着它才能访问受保护的接口。

---

## 一、先建立两个概念

| 概念 | 一句话 | 本步对应 |
| --- | --- | --- |
| **认证 Authentication** | 你是谁？——核对身份 | 注册存密码、登录比对密码 |
| **鉴权 Authorization** | 你能做什么？——检查权限 | 带 token 才能访问 `/api/users`、`/api/auth/me` |

前端最熟的类比：**认证 = 登录页输账号密码**；**鉴权 = 路由守卫**（`router.beforeEach` / `<PrivateRoute>`，没登录就把你挡在受保护页面外）。后端做的是同一件事，只不过守的是 HTTP 接口。

---

## 二、两块基石

### 1. 密码加密：为什么不能存明文

铁律：**密码永远只存「哈希值」，绝不存明文。** 数据库一旦泄露，明文密码等于直接送人，还会连累用户在别处复用的同一个密码。

我们用 **bcrypt**（`src/lib/password.ts`），它有三个关键特性：

- **故意慢**：算一次要花点时间。md5 一秒能算上亿次，bcrypt 一秒只算几千次 —— 暴力破解的代价被拉到天上。
- **自带 salt（盐）**：每次哈希都掺一段随机盐，所以**两个人用同样的密码，存进库的哈希也完全不同**，攻击者没法用「彩虹表」批量反查。
- **盐和成本参数都编码在结果里**：验证时 bcrypt 自己能读出来，我们无需额外存盐。

存进库长这样：`$2b$10$2YaGaXSIEeuq...` —— `$2b` 是算法、`$10` 是成本因子、后面是盐+哈希。

```ts
export function hashPassword(plain: string) {
  return bcrypt.hash(plain, 10); // 10 = 成本因子，每 +1 计算量翻倍
}
export function verifyPassword(plain: string, hash: string) {
  return bcrypt.compare(plain, hash); // 返回 true/false，绝不「解密」——哈希不可逆
}
```

### 2. JWT：一张可自证真伪的门票

**JWT（JSON Web Token）** 是服务端签发的一张「盖了防伪章的门票」（`src/lib/jwt.ts`）。登录成功后发给你，之后每次请求你都带上它，服务端验一下章是不是自己盖的，就知道你是谁 —— **全程不用查数据库存 session，这就是「无状态鉴权」**。

一个 token 三段，用 `.` 隔开：`header.payload.signature`

- **header**：算法信息
- **payload**：我们放的数据（这里是 `sub`=用户 id、`email`）。⚠️ **只是 base64 编码、不是加密**，任何人都能解开看内容 —— 所以**绝不往里放密码等敏感信息**。
- **signature**：用 `JWT_SECRET` 对前两段签名。改动 payload 会导致签名对不上 → 验签失败。「防伪」就靠这个：没有密钥就伪造不出合法签名。

```ts
signToken({ sub: user.id, email: user.email }); // 登录时签发
verifyToken(token);                              // 每次请求时验签+验过期，失败抛 401
```

`JWT_SECRET` 是核心机密（谁拿到它就能伪造任意用户的 token），所以纳入了 `env.ts` 校验、**不给默认值**（配错就 fail fast），并要求至少 16 位。

## 三、完整链路

沿用前面几步搭好的分层与基础设施（zod 校验、统一响应、全局错误处理、错误类），auth 也是一条干净的全链路：

```
路由 auth.routes.ts
  → validate 中间件（zod 校验 body）
  → controller（HTTP 翻译）
  → service（哈希/比对/签发，纯业务）
  → Prisma（读写 users 表）
```

三个端点：

| 方法 | 路径 | 说明 | 是否受保护 |
| --- | --- | --- | --- |
| POST | `/api/auth/register` | 注册（注册即登录，直接返回 token） | 公开 |
| POST | `/api/auth/login` | 登录，返回 token + user | 公开 |
| GET | `/api/auth/me` | 返回当前登录用户（token 解出的 payload） | 🔒 需 token |

成功响应统一是 `{ success, data: { token, user } }`，其中 **user 永远不含 password**。

## 四、鉴权中间件：路由守卫

`src/middlewares/authenticate.ts` 就是那个「门卫」：

```ts
export function authenticate(req, _res, next) {
  const header = req.headers.authorization;              // 约定：Authorization: Bearer <token>
  if (!header?.startsWith('Bearer ')) throw new UnauthorizedError('缺少 ... 请求头');
  const token = header.slice(7).trim();
  req.user = verifyToken(token);                          // 验签通过就把用户挂到 req.user
  next();                                                 // 放行
}
```

两种用法：

```ts
authRouter.get('/me', authenticate, getMe);  // ① 给单个路由加保护
userRouter.use(authenticate);                // ② 给整个 router 加保护（其后所有路由都要 token）
```

Step 9 起，**整个 `/api/users` 资源都受保护**（`userRouter.use(authenticate)` 一行搞定）。

### `req.user` 的类型从哪来

Express 官方的 `Request` 类型里没有 `user` 字段，直接写 `req.user` 会报类型错。我们用 TS 的**声明合并**（`src/types/express.d.ts`）给它「叠加」一个可选属性，全项目的 `req.user` 就都有类型了 —— 没改任何源码，只是往已有类型上补声明。

## 五、两个安全细节（面试常问）

1. **登录失败别泄露「邮箱到底注册没注册」**：无论是「邮箱不存在」还是「密码不对」，都回同一句含糊的 401「邮箱或密码不正确」。否则等于帮攻击者枚举哪些邮箱已注册。
2. **用白名单挑选可返回字段，而不是「删掉 password」的黑名单**（`auth.service.ts` 的 `toSafeUser`）：以后 User 若新增敏感字段（如 `resetToken`），只要不主动加进白名单就绝不会泄露 —— **默认安全**，比「记得删」可靠得多。读用户列表/详情则用 Prisma 的 `omit: { password: true }` 从查询层就排除密码。

## 七、验证（全部通过）

```bash
docker compose up -d && pnpm db:migrate   # 起库 + 应用迁移
pnpm dev

# 注册 → 201，返回 token + user（无 password）
curl -X POST localhost:3000/api/auth/register -H 'Content-Type: application/json' \
  -d '{"email":"alice@example.com","password":"secret123","name":"Alice"}'

# 无 token 访问受保护接口 → 401
curl localhost:3000/api/users

# 带 token → 200
curl localhost:3000/api/users -H "Authorization: Bearer <token>"
```

| 场景 | 期望 | 结果 |
| --- | --- | --- |
| 注册 | 201 + token + user（无 password） | ✅ |
| 无 token 访问 `/users` | 401 | ✅ |
| 错误 token | 401 | ✅ |
| 带 token 访问 `/me`、`/users` | 200 | ✅ |
| 重复注册 | 409 | ✅ |
| 登录正确 / 错密码 | 200 / 401（含糊提示） | ✅ |
| 弱密码注册 | 400（zod 校验） | ✅ |
| 库中 password 字段 | bcrypt 哈希 `$2b$10$...`，非明文 | ✅ |
