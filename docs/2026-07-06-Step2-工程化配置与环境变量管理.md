# Step 2 · 工程化配置与环境变量管理

> 目标：接入 ESLint + Prettier 统一代码质量与风格；建立「校验 + 带类型」的环境变量管理体系（后端核心基本功）。

## 本步做了什么

1. 安装代码质量工具：`eslint`、`@eslint/js`、`typescript-eslint`、`prettier`、`eslint-config-prettier`、`globals`
2. 安装运行时依赖：`dotenv`（加载 .env）、`zod`（校验）
3. `eslint.config.js`：ESLint 9/10 的扁平配置
4. `.prettierrc.json` / `.prettierignore`
5. `src/config/env.ts`：环境变量的唯一入口（校验 + 转换 + 类型）
6. `.env`（本地，不提交）与 `.env.example`（模板，提交）
7. `src/index.ts` 改用 `env.PORT`，不再直接读 `process.env`
8. `package.json` 增加 `lint` / `lint:fix` / `format` / `format:check` 脚本

## 概念讲解（含前端类比）

### 1. ESLint vs Prettier —— 分工

- **ESLint** 管**代码质量**（未使用变量、可能的 bug、类型问题）。
- **Prettier** 管**代码格式**（引号、缩进、换行、逗号）。
- 两者会在「格式」上冲突，所以 `eslint-config-prettier` 放在 ESLint 配置**最后**，关闭所有格式类规则，让 Prettier 独占格式。

> 前端类比：和你在 React 项目里用的完全是同一套思路。唯一新东西是 **flat config**（`eslint.config.js` 数组式配置），它是取代老 `.eslintrc` 的新标准。

### 2. 环境变量：后端的关键基本功

**为什么不把配置写死在代码里？**

- 密钥（数据库密码、JWT 密钥）写进代码 = 泄密 + 进 git 历史删不掉。
- 开发 / 测试 / 生产环境的配置不同，写死就没法切换。

**标准做法**：配置放环境变量 → 代码从环境变量读。

**环境变量的两个坑，以及我们怎么解决：**

| 坑                                           | 解决                                         |
| -------------------------------------------- | -------------------------------------------- |
| 全是字符串（`PORT` 是 `"3000"` 不是 `3000`） | `z.coerce.number()` 自动转换                 |
| 漏配 / 配错，运行时才崩、难排查              | 启动时用 zod 校验，配错立刻退出（fail fast） |

### 3. `src/config/env.ts` 的三大价值

1. **校验**：`safeParse` 检查所有变量；失败就打印每条问题并 `process.exit(1)`。**服务宁可启动不了，也不要带着错误配置裸奔**。
2. **类型**：导出的 `env.PORT` 是 `number`、`env.NODE_ENV` 是 `'development' | 'production' | 'test'` 联合类型，全程自动补全，不再是 `any`。
3. **转换**：字符串 `"3000"` → 数字 `3000`。

> 铁律：**全项目只在这一个文件里读 `process.env`，其它地方一律 `import { env }`。** 这样配置有单一可信来源，改一处即可。

### 4. `.env` vs `.env.example`

- `.env`：你本地真实配置，可能含密钥，**被 gitignore，绝不提交**。
- `.env.example`：模板，**提交进仓库**。他人克隆后 `cp .env.example .env` 就知道要配哪些变量。团队协作惯例。

### 5. 一个 ESM 小坑：import 要写 `.js`

在 `index.ts` 里写的是 `import { env } from './config/env.js'`——注意是 `.js` 不是 `.ts`。因为 tsconfig 用了 `NodeNext` 模块解析，遵循 ESM 规范：**import 路径要写编译后的产物扩展名**。源码是 `.ts`，但引用时写 `.js`。刚开始会不习惯，记住即可。

## 动手验证

```bash
pnpm typecheck        # 类型检查
pnpm lint             # 代码质量（无输出=通过）
pnpm format           # 自动格式化全部文件
pnpm format:check     # 只检查不改

# 验证 fail-fast：故意传非法端口，服务应拒绝启动并 exit 1
PORT=abc pnpm exec tsx src/index.ts
# → ❌ 环境变量校验失败： - PORT: Invalid input: expected number...

# 正常启动
pnpm dev
curl http://localhost:3000/health
```
