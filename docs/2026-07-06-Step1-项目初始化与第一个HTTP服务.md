# Step 1 · 项目初始化与第一个 HTTP 服务

> 目标：搭好 TypeScript 工具链，用 Node 原生 `http` 模块跑通第一个 Web 服务器，理解「服务器的本质」。

## 本步做了什么

1. `pnpm init` 初始化 `package.json`
2. 安装工具链：`typescript`、`@types/node`、`tsx`
3. 写 `tsconfig.json`（TypeScript 编译配置）
4. 写 `.gitignore`
5. 写 `src/index.ts`（用原生 http 模块的第一个服务器）
6. 配置 `package.json` 的 scripts

## 涉及文件

```
├── src/index.ts        # 服务入口

```

## 概念讲解（含前端类比）

### 1. package.json —— 和前端几乎一样

| 字段 | 作用 | 前端类比 |
|---|---|---|
| `"type": "module"` | 用现代 `import/export`（ESM）而非老的 `require` | Vite 项目默认 ESM |
| `scripts.dev` = `tsx watch` | 改代码自动重启服务 | 前端的 `vite` 热更新 |
| `scripts.build` = `tsc` | 把 `.ts` 编译成 `.js` 到 `dist/` | 前端的打包 |
| `scripts.start` = `node dist/index.js` | 用 Node 跑编译产物 | —— |
| `scripts.typecheck` = `tsc --noEmit` | 只做类型检查、不产出文件 | —— |

**关键区别**：前端产物给浏览器跑；后端产物在服务器上由 **Node 进程**跑，且进程**常驻不退出**（一直等请求）。

### 2. tsconfig.json —— 后端 TS 和前端的不同

- `"lib": ["ES2022"]`：**没有 `"DOM"`**！后端没有 `window`、`document`。少写这个会导致 `console` 报错。
- `"types": ["node"]`：引入 Node 类型（`process`、`http`、`fs`…）。用 pnpm 时通常需要显式声明，否则 `@types/node` 不被自动发现。
- `strict: true` + `noUnusedLocals` 等：企业级必开，编译期抓 bug。
- `rootDir: src` / `outDir: dist`：源码放 `src`，编译产物进 `dist`。

### 3. src/index.ts —— 服务器的本质

没用任何框架，只做三件事：

```
http.createServer(回调)   →   每来一个请求执行一次回调
        ↓
   回调里看 req.method + req.url，决定返回什么
        ↓
   server.listen(端口)     →   挂起进程，永久监听
```

- **req（请求对象）**：谁来的、什么方法（GET/POST）、访问哪个路径、带了什么数据。
- **res（响应对象）**：往里写状态码 → 响应头 → 响应体，`res.end()` 后发回对方。
- 这套 `req/res` 模型 **Express 完全沿用**，框架只是在外面包了更好用的 API。

### 4. 为什么第一步故意不用框架？

手写 `if (url === '/health')` 就要一堆 `writeHead` / `JSON.stringify`。50 个接口全手写会累死。Step 3 引入 Express 后，你就能精确感受到框架替你解决了什么。

### 5. 健康检查接口（/health）

企业标配。负载均衡 / K8s 靠它判断服务是否存活，返回 `200` 表示健康。

## 动手验证

```bash
pnpm typecheck                        # 类型检查，应输出通过
pnpm dev                              # 启动（改代码自动重启）
curl http://localhost:3000/health     # 另开终端访问 → {"status":"ok",...}
curl http://localhost:3000/foo        # 不存在路径 → {"message":"Not Found"}
```

## 小结

服务器 = 监听端口 + 处理 `req` 生成 `res`。TypeScript 工具链就位，`dev/build/start` 三件套跑通。下一步进入工程化配置（ESLint / Prettier / 环境变量）。
